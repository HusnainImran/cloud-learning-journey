## Commands Learned & Practiced:

### 1. Process Viewing (ps)
- **ps** - Show processes
  - `ps` - Show your current terminal processes
  - `ps -a` - Show all processes with a terminal
  - `ps -u` - Show processes with user information
  - `ps -x` - Show processes without a terminal
  - `ps aux` - Show ALL processes with detailed info (most common)
  - `ps -ef` - Show all processes in full format
  - `ps -f` - Full format listing
  - `ps l` - Long format with more details
  - `ps a` - Show all processes on terminal

### 2. Process Filtering
- **Piping with grep** - Find specific processes
  - `ps aux | grep cat` - Find processes containing "cat"
  - `ps aux | less` - View all processes page by page

### 3. Process IDs
- **echo $$** - Show current shell's PID
- **pgrep** - Find process ID by name
  - `pgrep ps` - Get PID of ps command
  - `pgrep ps -l` - Show PID with process name (attempted)

### 4. Interactive Process Monitoring
- **top** - Real-time process monitor
  - Shows CPU, memory usage
  - Updates continuously
  - Press 'q' to quit
- **htop** - Enhanced interactive process viewer
  - Installed with: `sudo apt install htop`
  - Better UI than top
  - More colorful and user-friendly

### 5. Process Tree
- **pstree** - Show process hierarchy
  - Installed with: `sudo apt install psmisc`
  - Displays parent-child process relationships

### 6. Killing Processes (Process Termination)
- **kill** - Terminate processes by PID
  - `kill 49480` - Default signal (SIGTERM)
  - `kill -15 49480` - Graceful termination (SIGTERM)
  - `kill -9 49480` - Force kill (SIGKILL)
  - `kill -9 31705 && kill -9 26585` - Kill multiple processes
- **kill -l** - List all available kill signals
- **killall** - Kill processes by name
  - `killall sleep` - Kill all processes named "sleep"

### 7. Background Jobs Management
- **&** - Run process in background
  - `sleep 300 &` - Run sleep for 300 seconds in background
  - `sleep 200 &` - Start another background process
  - `sleep 100 &` - Start third background process
- **jobs** - List background jobs
- **fg** - Bring job to foreground
  - `fg` - Bring most recent job to foreground
  - `fg %1` - Bring job number 1 to foreground
- **bg** - Continue job in background (after Ctrl+Z)

### 8. System Commands
- **exit** - Exit terminal/shell session

## Key Learnings:

### 1. Understanding ps Command Variations
- **ps** alone shows very limited info (just current terminal)
- **ps aux** is the most comprehensive (shows ALL processes)
- **ps -f** shows full format with PPID (parent process ID)
- Different flags give different information levels

### 2. Process States & Information
When using `ps aux`, I learned:
- **USER** - Who owns the process
- **PID** - Process ID (unique identifier)
- **%CPU** - CPU usage percentage
- **%MEM** - Memory usage percentage
- **COMMAND** - The actual program running

### 3. Kill Signals Hierarchy
- **kill PID** - Default SIGTERM (signal 15) - Polite request to stop
- **kill -15 PID** - Same as above (graceful shutdown)
- **kill -9 PID** - SIGKILL - Forces immediate termination (cannot be ignored)
- Use -15 first, -9 only if process won't stop

### 4. Background vs Foreground
- **command &** - Starts in background
- **Ctrl + Z** - Pauses current foreground process
- **bg** - Resumes paused process in background
- **fg** - Brings background process to foreground
- **jobs** - Lists all background jobs

### 5. Tools Comparison
- **ps** - Snapshot of processes at one moment
- **top** - Continuous real-time monitoring (updates every few seconds)
- **htop** - Enhanced top with better UI, colors, easier navigation
- **pstree** - Shows parent-child relationships

### 6. Process Termination Methods
Two ways to kill processes:
1. **By PID**: `kill 1234` (need to know exact PID)
2. **By Name**: `killall processname` (kills ALL matching processes)

## Experiments & Discoveries:

### Experiment 1: Testing Invalid ps Combinations
```bash
ps aux -ef            # Tried combining aux and ef flags
```
**Result:** Error - can't combine BSD style (aux) with UNIX style (-ef)
**Lesson:** ps has two syntax styles, can't mix them

### Experiment 2: Finding Process IDs
```bash
echo $$               # Shows my shell's PID
pgrep ps              # Attempted to find ps command PID
```
**Lesson:** `$$` is useful for scripting to know current shell PID

### Experiment 3: Multiple Kill Attempts on Same Process
```bash
kill 49480            # First attempt
kill -15 49480        # Second attempt
kill -9 49480         # Final force kill
```
**Lesson:** Some processes need force kill (-9) when they don't respond to graceful termination

### Experiment 4: Chaining Kill Commands
```bash
kill -9 31705 && kill -9 26585
```
**Discovery:** Can kill multiple processes in one line using `&&`

## Questions Answered During Practice:

**Q: What's the difference between ps and ps aux?**
A: `ps` shows only my processes, `ps aux` shows ALL system processes

**Q: When to use kill -9 vs regular kill?**
A: Try regular `kill` first (graceful), use `kill -9` only when process won't stop

**Q: What does & at the end of command do?**
A: Runs the command in background, gives you terminal back immediately

**Q: How to bring background job to foreground?**
A: Use `fg` for most recent, or `fg %1` for specific job number
